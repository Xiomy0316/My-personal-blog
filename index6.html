Tutorial de Git

Descripción

Aprende paso a paso cómo usar la terminal de Git para trabajar en tu proyecto.

¿Qué es Git?

Git es un sistema de control de versiones distribuido, lo que significa que un clon local del proyecto es un repositorio de control de versiones completo. Estos repositorios locales plenamente funcionales permiten trabajar sin conexión o de forma remota con facilidad.

En un sistema de control de versiones distribuido, cada desarrollador tiene una copia integral del proyecto y del historial del mismo, proporcionándoles una vista consistente y unificada a los miembros del equipo de desarrollo.

Planeación

El primer paso para usar Git es descargarlo desde su página oficial a nuestro equipo, este ofrece diferentes versiones y permite descargas para macOS, Windows y Linux/Unix. Vamos a nuestra carpeta de descargas y abrimos nuestro archivo de Git, te pedirá permitir que esta aplicación haga cambios en el dispositivo a lo que se le debe dar aceptar, luego, te saldrán diferentes ventanas de diálogo, a las que le darás next o siguiente, dejando cada cosa por defecto, hasta que te aparezca la ventana de diálogo que pide seleccionar el editor de código, para este caso seleccionaremos Use Visual Studio Code as Git´s default editor, lo que significa Utilice Visual Studio Code como editor predeterminado de Git; le damos next o siguiente nuevamente a cada ventana de diálogo que nos aparezca, dejando todo por defecto, hasta que nos salga el botón install o instalar en la parte inferior del cuadro de diálogo, le damos clic y esperamos a que se complete la instalación, para luego presionar finish o finalizar.

Después de haber descargado Git en nuestro equipo, procedemos a abrir nuestra terminal de Git, buscando la aplicación a través de la barra de búsqueda con la palabra Git o directamente en nuestro equipo.

Configuración inicial

Antes que todo debemos asegurarnos que Git esté correctamente instalado en nuestro sistema. Puedes verificarlo con git --version o git -v que sería la abreviación.

Luego, debemos configurar nuestro nombre y correo electrónico, con los comandos git config --global user.name “Nombre de usuario” y git config --global user.email “Correo electrónico”.

Por ejemplo, git config --global user.name “Pepe Pérez” y git config --global user.email “pepitoperez@gmail.com”.

Por último, se debe comprobar que se han establecido las configuraciones, para ello ejecutaremos el comando git config --list.

Inicialización del repositorio localmente

Para comenzar, debemos saber en que lugar de nuestro equipo nos encontramos actualmente, con el propósito de definir si es el lugar deseado para albergar nuestro proyecto, o si por el contrario, queremos cambiar la ruta donde se guardará este localmente. Para ello ejecutaremos el comando pwd como se muestra a continuación.

Si necesitamos cambiar la ruta donde se albergará nuestro proyecto, simplemente ejecutamos cd /el lugar donde queremos albergarlo. Ejemplo: cd /d.

Luego, enlistamos todos los directorios o carpetas, y archivos que contiene el lugar que seleccionamos para albergar nuestro proyecto, en este caso el disco d; para esto, ejecutamos el comando ls.

Para crear un directorio o carpeta, el cual contendrá nuestro proyecto, ejecutaremos el comando mkdir, seguido del nombre de la carpeta o directorio que queremos crear. Ejemplo: mkdir Tutorial_Git.

Nuevamente colocamos ls para enlistar, ya allí nos debe de aparecer la nueva carpeta o directorio que creamos, en este caso, Tutorial_Git.

Ahora ingresamos a la carpeta o directorio, para lo cual colocamos cd, seguido del nombre de la carpeta. Ejemplo: cd Tutorial_Git.

Una vez que estamos dentro de la carpeta vamos a crear un archivo, para ello colocamos touch seguido del nombre del archivo que queremos crear, en este caso index.html

Para confirmar que el archivo ha sido creado de forma exitosa enlistamos con ls; allí nos deben aparecer los archivos creados, para este caso index.html

A continuación, abrimos el Visual Studio Code con el comando code .

Después de abrir Visual Studio Code, procedemos a crear nuestro repositorio Git con el comando git init, el cual nos va a permitir mantener las versiones de nuestro código.

Si queremos corroborar que el paso anterior fue ejecutado correctamente, simplemente colocamos ls -a, lo que permitirá que se muestren los directorios ocultos.

Luego de haber seguido estos pasos, procedemos a mirar el estado de nuestro repositorio con git status, donde sabremos en que etapa se encuentra nuestro proyecto.

Como vemos, nos encontramos actualmente en la ramificación master o rama principal; no se encuentra ningún commit todavía y hay un archivo de nombre index.html que aún no está siendo rastreado. Para que Git rastree este archivo, lo debemos agregar o adicionar y para ello, usaremos el comando git add seguido del nombre del archivo. Ejemplo: git add index.html

Nota: lo ideal es hacer este procedimiento por cada archivo no rastreado o ya modificado, ya que se considera una buena práctica, pues también tenemos la posibilidad de hacer una adición de forma global o, mejor dicho, agregar todos los archivos a la vez, esto ocurre con el comando git add seguido de un punto. Ejemplo: git add .

Nuevamente colocamos git status, para verificar el estado del repositorio, donde nos muestra que el archivo está siendo rastreado ahora (está agregado) y se encuentra en fase de staged.

Ahora procederemos a hacer el envío al repositorio local de los cambios realizados, para esto usaremos el comando git commit -m “comentario de los cambios que hice para este staged”.
